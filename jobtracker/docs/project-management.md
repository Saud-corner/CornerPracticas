# Gestión del Proyecto

Para la organización de este proyecto he decidido utilizar la metodología **Kanban**. 

Como es un proyecto individual y necesito flexibilidad, Kanban me permite visualizar el flujo de trabajo sin las restricciones de tiempo estrictas que impone Scrum con los Sprints.

Para ello, he creado un tablero en **Trello** con las siguientes columnas:
* **Backlog:** Donde apunto todas las funcionalidades que pide la rúbrica y las ideas futuras.
* **Todo:** Las tareas que voy a programar de forma inminente.
* **In Progress:** Las tareas en las que estoy trabajando en este momento (para no saturarme, intento tener solo 1 o 2 tarjetas aquí a la vez).
* **Review:** Tareas terminadas que necesitan una última revisión de código o pruebas antes de darlas por buenas.
* **Done:** Funcionalidades completadas y subidas a GitHub.

Cada tarjeta grande (ej. "Crear Base de Datos") la divido en subtareas técnicas más pequeñas para ir marcando casillas y ver el progreso real.
